Cost-Effective TB-Scale Memory Hierarchy for Next-Generation Datacenter

With DRAM having slowed down in density scaling, the only path forward to improving density in servers is a tighter integration of denser technologies. Storage-class memory has the potential to improve density but has been on the horizon for a while and is likely to maintain a high cost in the near future. Solid State Disks (SSDs) offer a 50x density advantage (and cost/capacity) but with also 500x (50us) longer access latency over DRAM. As such, SSDs have historically been treated as storage devices connected as a peripheral and interfaced through a host OS. Because for many online services the end-to-end tail latency requirement is within hundreds of millisecond, we believe that a careful hardware/software co-design for a tighter integration of SSDs into a host CPU and DRAM cache would enable a substantial reduction in cost with no impact on throughput while maintaining end-to-end tail latency. We indicate a 32x reduction in DRAM capacity in a cache allowing for an overall 20x reduction in memory cost with an SSD.

Current proposals for a tighter integration of SSDs argue for byte addressability. Unfortunately, the latter relies on conventional paging and address translation through host OS to promote pages between the SSD and DRAM which results in high swap overhead that takes up to 100â€™s of microseconds. Instead, we propose to investigate SSDs that are directly mapped into the address space with address translation support in hardware and a DRAM cache. We propose to investigate DRAM cache architectures that allow for high-bandwidth lookup by CPUs with custom support for miss handling. Due to the high latency of SSD access, we will investigate new miss handlers that would manage the DRAM cache as a page-based cache. Because of the high frequency of invoking such handlers, the traditional host OS swapping mechanisms are prohibitively expensive. Moreover, there is no need for TLB shootdowns or placing an eviction/writeback on the critical path of a page read. We will investigate hardware/software co-design (outside of a host OS) to manage a DRAM cache with a mapped SSD.
